Test Client and Scenarios


usage: Tangerine Test Client

 -collection <arg>      Collection to use
 -collections           Show all collections names
 -configuration <arg>   Reasoner configuration file. Required for
                        reasoner.
 -count                 Count documents in collection. (Use with
                        collection)
 -drop                  Drop collection
 -elements <arg>        Show X amount of documents. (Use with collection)
 -entity <arg>          entity to use.(Optional. Use with insert and uuid)
 -help                  Show this message
 -hostname <arg>        Server host name or IP.
 -insert <arg>          Insert data. If the collection already exist it
                        will append to it. (Use it with collection and
                        type)
 -ontologies <arg>      Ontology file. Required for reasoner.
 -output-type <arg>      reasoner output (csv, json[default])
 -port <arg>            Port number of the service
 -queries <arg>         Reasoner queries file. Required for reasoner.
 -reasoner              Send reasoner query. (Use with configuration,
                        ontologies and queries)
 -type <arg>            Type of data that is going to be use (netowl,
                        voucher, pii, esri, general[default])
 -uuid <arg>            uuid to use when inserting data.(Optional. Use
                        with insert and entity)


For the following test scenarios, assume the following base command line:


	java -jar ./client.jar (args)

	where CLASSPATH is configured in JAR to use the local path "./lib/*jar"
	and the Java main class is TangerineTester;  
	We are always staring in the test client install folder.
	Finally, we assume you are working on localhost. Insert the valid server name if not localhost.


Use Case #1 Test Scenario

This quick summary of uses of the test client only show command line invocation for discrete operations.  To make more sense of how to apply this in a particular use case, consult Use Case #1 demonstration page
// Insert data.  Capture the reported "collection" ID as P1.  
// We may use this variable to refer to the collection rather than a literal string

TangerineTester -hostname localhost -insert ../demo/PII.csv  -type pii
// Insert more data.  Capture the reported "collection" ID as V1

TangerineTester -hostname localhost -insert ../demo/Test-Claims.csv  -type voucher

// List the collections;  Confirm you see P1 and V1 in the list.
TangerineTester -hostname localhost -collections

// Query the collection for a row count and get actual elements, aka raw data or assertions.
TangerineTester -hostname localhost -collection P1  -count

// 
TangerineTester -hostname localhost -collection P1  -elements 100
